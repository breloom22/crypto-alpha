# Crypto Alpha — Strategy Discovery & Optimization

A look-ahead-safe research pipeline that **discovers, validates, and optimizes OHLCV trading
strategies** across a 9-asset crypto universe — plus a companion **crash-detector** study. Built
around a hard overfitting defense: every result must survive in-sample/out-of-sample, walk-forward,
and a random-entry significance test before it is believed.

> **Timeframe: daily (1d) bars.** All holding periods, cooldowns, and indicators are in *days* —
> this is a swing-trading study, not intraday. Universe: BTC, ETH, SOL, DOGE, OP, AVAX, XRP, XLM,
> SUI · 2022-01-01 → 2026-06-06 · entry filled at the **next day's open** after a signal.

| Phase | What | Headline result |
|------|------|-----------------|

## The pipeline, phase by phase

### v1 — Crash Detector (`main.py`)
Objective crash-leg definition (7d ≤ −15%, 3d ≤ −10%, drawdown ≥ 20%, 1d ≤ −8%; onset-edge,
14-day clustering) → 35 leading-indicator candidates → leakage-safe backtest → precision/recall/F1
vs a **base rate ≈ 0.39** (crypto crashes are frequent, so *lift over base rate* is the honest
metric) → ensembles → in/out-sample + sensitivity → 7-chart dashboard.
**Finding:** only momentum-exhaustion signals (Stochastic cross >80, Williams %R, CCI reversal)
clear the base rate (~1.2–1.3×, ~10-day lead). High-recall signals look good on F1 but add no edge.

### v3 — Strategy Discovery (`main_v3.py`)
Strategies are composed from independent blocks and generated by the hundreds:

```
Strategy = Signal (S1–S8, 102 signals) × Direction (LONG/SHORT) × Exit (E1–E12) × Filter (F1–F10)
```

Five generation layers (single → exit variants → filters → AND-combos → SEQ long/short pairs),
then ranked by a composite score and pushed through the **5 validation gates** (below). Of 1,164+
candidates, 17 survived OOS, and one — `SEQ_S1.11_S+S4.04_L` (NR7 short + Normalized-ATR reversal
long) — passed **all five gates**.

### v4 — Mechanism-based Optimization (`main_v4.py`)
Takes the 7 validation-passing strategies and improves their **trade-level behavior without
touching entry signals**, via 7 independent modules:

| | Module | | | Module | |
|--|--------|--|--|--------|--|
| **A** | ATR dynamic SL/TP | | **E** | alt LONG leg for SEQ |
| **B** | break-even + extended hold | | **F** | stepped ATR trailing |
| **C** | whipsaw/chop guards | | **G** | multi-strategy portfolio |
| **D** | asset selection / weighting | | | |

154 variants are scored on IS, gated, OOS-compared to the original, then winners get walk-forward +
random. **Module effectiveness was honest and surprising:** F (+5.6pp) and A helped; adding filters
(C, −15.8pp) and swapping signals (E) mostly *hurt* on OOS. 5/7 strategies improved; 2 kept their
original. `--compare` / `--portfolio` / `--module A` for partial runs.

### v5 — Per-strategy Parameter Tuning (`main_v5.py`)
A bespoke **staged (greedy) grid** per strategy — entry timing, signal periods, SL/TP, hold,
trailing, asset exclusion, loss-streak cooldown — searched on IS, with the IS-gated top-10 validated
on OOS. A point is adopted only if it beats **both** the v3 original and the v4 best on OOS.
**1/5 won** (`SEQ_S1.11_S+S3.01_L` → +75.9%, WF 4/5, p=0.000, by swapping its dormant Supertrend
long for a tuned NATR-reversal long). The other four scored higher on IS but failed the OOS gate —
a clean demonstration of IS over-fitting being filtered out. It also confirmed the project's
"0~1 day whipsaw" diagnosis: the winner lifts 0-day win-rate from 21% → 57%.
`--strategy <id>` / `--stage` (IS only) / `--compare`.

## The 5-gate validation framework (overfitting defense)

| Gate | Criterion | Meaning |
|------|-----------|---------|
| ① Survived | OOS total return > 0 | doesn't lose money out-of-sample |
| ② Valid | OOS avg profit factor > 1.0 | wins outweigh losses |
| ③ Robust | OOS Sharpe > 0.5 × IS Sharpe | edge persists, not just IS |
| ④ Walk-forward | ≥ 3 of 5 rolling windows OOS-positive | stable across time |
| ⑤ Random | beats 95th pctile of 1,000 random-entry sims (p<0.05) | edge isn't luck |

IS window 2022-01 → 2024-06 · OOS window 2024-07 → 2026-06. *Most* strategies fail ④/⑤ — that is the
defense working, not a bug.

## Backtest engine (no look-ahead by construction)

- **Entry** = bar *after* a signal, filled at that bar's **open**. **Exit** = condition bar's close,
  except intraday SL/TP/trailing/ATR/chandelier stops which fill at their trigger (checked via High/Low).
- Same-bar **SL beats TP** (conservative); trailing stops trail the peak through the **prior** bar only.
- Slippage **0.1%/side**, non-overlapping positions, 3-bar cooldown, ≥10 trades/cell to count,
  returns summed **non-compounded**.
- v4/v5 **extend the v3 engine non-destructively** — v3 reproduction is verified bit-identical after
  every addition (ATR/trailing/break-even/loss-streak/cooldown all default to v3 behavior when unset).
- **Look-ahead safety** is enforced by a truncation-invariance test: dropping the last N bars must
  not change any earlier signal value (`src/_test_indicators.py`).

## Caveats

- **Research, not a trading system.** Slippage 0.1%/side only — no exchange fees, funding, or market
  impact. Returns are non-compounded sums of per-trade %.
- **Daily bars.** "0-day holds" are same-bar intraday SL/TP fills judged on the day's High/Low, not
  1-minute paths; the SL-beats-TP tie-break is a conservative assumption.
- **Overfitting / multiple testing** is the central risk; read every headline number alongside its
  walk-forward and random-benchmark columns. Past performance does not predict the future — paper-
  trade before risking capital.
- **Listing dates:** OP (2022-06) and SUI (2023-05) have shorter histories; this data has no overnight
  gaps (`open[t] == close[t-1]`), so the gap-fill signal never fires here.
